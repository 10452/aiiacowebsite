Round 112 — Admin Transcript Optimization

Architecture:
- WebTranscriptsTab in AdminConsolePage.tsx (lines 514-664)
- Uses trpc.talk.listTranscripts.useQuery() → getAllWebTranscripts()
- Transcript JSON format: Array<{role: 'user'|'ai', text: string, timestamp: string}>
- Also has transcriptText (plain text fallback)
- Fields: id, leadId, visitorName, visitorEmail, visitorPhone, transcript, transcriptText, durationSeconds, source, createdAt

Current UX:
- Expandable rows with inline transcript (maxHeight: 400px, simple text format)
- No chat bubbles, no live mode
- Search by name/email/phone/transcript text

Plan:
1. Replace the expanded transcript view with proper chat bubbles (matching AdminLeadsPage style)
   - AiA messages: gold-tinted bubbles, left-aligned
   - Visitor messages: blue-tinted bubbles, right-aligned
   - Timestamps on each message
   - No maxHeight limit — full scrollable view
   - Copy transcript button

2. Add a "Live Sessions" indicator at the top
   - Poll listTranscripts with refetchInterval: 5000 when live mode is on
   - Show a pulsing green dot for "recently active" transcripts (created in last 5 min with no durationSeconds)
   - When a "live" transcript is selected, auto-scroll to bottom as new messages appear
   - Animated entrance for new messages

3. Add a dedicated full-page transcript viewer (slide-out panel)
   - Click "View Full" to open a side panel with the complete conversation
   - Proper chat bubble layout
   - Visitor info header
   - Duration, date, message count stats
